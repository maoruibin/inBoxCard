## 目标
- 在应用内提供显眼、易达的“参与贡献”入口，展示 README 贡献流程，降低新贡献者的上手门槛。

## 入口位置
- 顶部导航：在右上角加入“贡献指南”按钮，点击跳转到指引页。
- 侧边栏：在分类区块下方新增“参与贡献”导航项，点击跳转到指引页。
- 首页引导：在首页标题下加入一条提示链接“想贡献卡片？查看贡献指南”。
- 详情页（原始视图）：在现有“查看详细图文教程”旁新增“贡献指南”外链，保持一致的样式。

## 页面内容（新建 `/contribute` 页面）
- 标题与简介：说明项目愿景与共建原则。
- 三步走流程（与 README 对齐）：
  1. 准备笔记文件（卡片时间戳模板 + 示例代码块，可一键复制/下载）
  2. 注册集合（`data.ts` 中添加示例配置代码块，可一键复制）
  3. 提交 PR（链接到仓库 `Pull Requests`，说明审核与发布）
- 常见问题：
  - 时间戳格式与解析器支持（`YYYY-MM-DD HH:mm[:ss]`，支持 `- . /` 分隔）
  - 标签格式（末尾 `#标签`，支持中文、斜杠）
  - 本地预览命令（`npm install`、`npm run dev`）
- CTA：
  - GitHub 仓库链接
  - “新建 Issue”链接
  - “下载模板文本”按钮（基于 Blob 生成 `.txt`）
